# DB 升级脚本说明

本目录以 `schema.sql` 与初始化脚本 (`docker-init.sql`, `init_admin.sh`, `seed_products.sql`) 为首次部署提供完整、可重复的数据库初始化。

对已部署的实例需要执行的一次性升级脚本（例如 ALTER TABLE 添加字段等）被移动到 `db/archive/`：

- `db/archive/`：存放针对历史部署/演进的迁移脚本，脚本文件内应包含运行说明和影响范围，适合由运维/管理员在维护窗口中手动执行。

维护原则：
- 新的功能如需改变数据模型，请优先在 `schema.sql` 中反映（用于新安装）。
- 已运行实例请在 `db/archive/` 查找相应的升级脚本并按注释执行，或使用数据库迁移工具将这些脚本转换为可自动化的迁移流程。
