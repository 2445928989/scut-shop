# 文档写作规范（简要）

目的：
- 保证仓库内的说明文档一致、易读、面向开发者与维护者。文档应直接、明确。

语气与风格：
- 使用第三人称或命令式：例如“运行 XX 命令以启动服务”，避免“我做了……”等表述。
- 保持简洁：每个章节不超过 4 段话，关键命令使用代码块展示。
- 明确步骤：必要时提供逐步指令与预期输出示例。
- 一致性：统一使用中文（变更记录、注释等可用英文），标题使用汉字与英文并列（如适用）。

格式：
- 使用 Markdown 标题（H1/H2/H3）组织内容。
- 代码示例使用三重反引号，并在语言标注中写明 shell / bash / docker 等。
- 对于重要命令，尽量包含在“快速开始”或“小节步骤”中。

文件级别元信息：
- 文档顶部应包含简短摘要与适用对象（例如：开发者、运维、测试）。
- 如果文档描述了命令或脚本，务必写明执行路径与前置条件（如环境变量、端口占用等）。

示例语句：
- 正确："在仓库根目录运行：`docker compose up -d --build`，服务启动后访问 http://127.0.0.1:3000。"
- 不推荐："我已经修改了…"。

维护建议：
- 文档改动应随功能变更同步更新，并在 PR 描述中列出对文档的修改点。
- 遇到调试记录或长日志，保留在 `doc/` 中并写简短总结到 README 中。
